В том случае если нам нужно отменить изменения сделанные в рамках
более раннего коммита мы можем использовать команду git revert <хеш комита>

// См. файл images/7.git_revert.jpg

Команда git revert <хеш комита> создаст новый комит с изменениями
противоположными тому коммиту который нужно отменить, таким образом мы не изменим
историю коммитов не получим ошибки синхронизации с remote origin.

git это система "Только вперед"
Правильный подход при работе с гитом — создание нового, а не изменение старого.
Тут речь идет о коммитах и тд.

Представим что мы несколько уомитов назад удалил file1.txt, но оказалось
что он нам нужен

~ git revert aa600a43cb164408e4ad87d216bc679d097f1a6c
# После этой команды откроется редактор, ожидающий ввода описания коммита
# Обычно сообщение revert не меняют, поэтому достаточно просто закрыть редактор
[main 65a8ef7] Revert "remove file1.txt"
 1 file changed, 1 insertion(+)
 create mode 100644 file1.txt
# В проект вернулся файл file1.txt
