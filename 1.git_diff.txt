Позволяет проверить какие именно файлы были изменеы и в каком месите

Создадим новый проект

project
    - file1.txt  // добавьте в него какой-то текст
    - file2.txt  // добавьте в него какой-то текст

Инициализируем git

> git init

и глянем статус

> git status

Видим что файлы не добавлены для отслеживания, добавим их

> git add * // завездочка позволяет добавить все текущие файлы для отследивания

> git status // файлы добавлены

теперь нам нужно изменить один из файлов, скажем file1.txt

// Изменили файл

Теперь мы хотим увидить в каком месте и какие строки были затронуты

> git diff

diff --git a/someFile.txt b/someFile.txt
index 0247eeb..1f0bad9 100644
--- a/file1.txt
+++ b/file1.txt
@@ -1 +1,2 @@
-asdas

// получим подробную инфу об изменении файлов всего прокта

hexlet-git$ git diff --staged
git diff — команда, которую нужно обязательно запускать перед каждым коммитом.
Она позволяет проанализировать добавляемые изменения и исправить возможные ошибки.
Иногда программисты по ошибке добавляют в коммит то, что туда не должно попасть.
